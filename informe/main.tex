\section{Third Problem (3.5 Points)}
Consider the following C program that contains a sort function, that it is not implemented, and it should sort an array of N integers. It also contains a function, checkSort, that it is intended to be used for testing, with assertions, that the sort function works as expected.
Per trobar la K, es necessita aquestes dues proves:
\begin{itemize}
    \item Amb K = 4 iteracions encara existeix una execució on el bucle es podria desenredar un cop més.
    
    \texttt{cbmc  --verbosity 8 --no-standard-checks --trace  --unwind 4 --unwinding-assertions  sort.c }
    
    \texttt{VERIFICATION FAILED, Violated property: file sort.c function checkSort line 33 thread 0 unwinding assertion loop 0}
    
    \item Amb K = 5 iteracions ja no existeix cap iteració on el bucle es desenredi aquest nombre de cops.
    
    \texttt{cbmc  --verbosity 8 --no-standard-checks --trace  --unwind 5 --unwinding-assertions  sort.c }
\end{itemize}

\begin{enumerate}
    \item Implement the sort function with your favourite sorting algorithm.
    \begin{lstlisting}
void sort (int8_t a[], int size) {
	 for (int i = 0; i < size - 1; i++)
    {

        for (int j = 0; j < size - i - 1; j++)
        {
            if (a[j] > a[j + 1])
                swap(a, j, j + 1);
        }
    }
}


void swap(int8_t *arr, int i, int j)
{
    int8_t temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
    \end{lstlisting}
    \item Write assertions at the end of the checkSort function to verify if your sorting algorithm works as expected. Check with cbmc the function checkSort, but with five different values of N: {4, 6, 8, 10, 12}. Write down the computation time needed by cbmc to verify the program with these four different array sizes. Do you think that the computation time increases exponentially ?
    
    \begin{table}[H]
        \centering
        \begin{tabular}{cccccc}
            N & 4 & 6 & 8 & 10 & 12 \\
            \hline
            time & 0.100s & 0.146 & 0.282 & 0.694 & 1.933 \\
        \end{tabular}
        \caption{Caption}
        \label{tab:placeholder}
    \end{table}
    \item Introduce an small error in the implementation of your sort function. Then, check it again (with the minimum size for N) to see if cbmc finds the error.
\end{enumerate}
